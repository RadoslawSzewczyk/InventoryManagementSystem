4. What frustrates you most about security tooling today? (2–3 sentences)
The exceptionally high rate of false positives often leads to alert fatigue, causing people to ignore critical warnings.
Additionally, many tools lack contextual understanding of the specific application architecture, making integration into modern,
fast-paced CI/CD pipelines overly rigid and noisy.

5. If you had a week to build anything combining AI + security, what would you build? 
I would build an AI code review agent that combines static and dynamic analysis.
It would ingest raw static analysis alerts and cross-reference them with runtime execution traces from dynamic
profiling tools like Valgrind. By correlating static code warnings with actual runtime memory behavior, the agent
could definitively filter out false positives, highlight the exact execution path of the vulnerability, and automatically
draft targeted mitigation PRs for confirmed memory safety issues.

8. What's your reaction when an AI tool produces a false positive? (2–3 sentences)
I treat it as a limitation of the current technology and an opportunity to refine my prompting or
enhance the context. I usually adjust the constraints, provide examples, or break down the task down into
smaller, tasks to guide the LLM back on track.
